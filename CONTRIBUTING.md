# Contributing to Awesome TLA

Thank you for your interest in contributing to Awesome TLA!

## Guidelines

- **Quality over quantity**: Only submit resources you have personally used or are widely recognized
in the TLA community
- **Provide context**: Include a brief description of what the resource is and why it&#39;s useful

- **Keep it current**: Resources should be actively maintained (within last 2 years for tools,
specifications may be older)
- **Follow the format**: Match the style and format of existing entries
- **One resource per pull request**: Makes review easier

## How to Contribute

1. Fork this repository
2. Create a new branch for your addition
3. Add your resource to the appropriate section in README.md
4. Commit your changes with a descriptive message
5. Push to your fork
6. Submit a pull request

## What to Include

Good additions include:
- Open-source tools and libraries
- Reference implementations
- Official specifications and standards
- Well-maintained documentation
- Active community resources

Please don&#39;t include:
- Commercial products without free tier or open-source version
- Unmaintained projects (last update &gt;2 years ago)
- Personal projects without community adoption
- Resources behind paywalls

## Questions?

Open an issue or ask in the [TLA Discussion Forum](https://discuss.tlaworks.com)
